1.      RELEASE DESCRIPTION & BACKGROUND

This release will implement the release for  Elig Feeds: iMAX: Decouple EOI from Elig code

2.      SCOPE

Testing will validate The EOI code is in the same DLL as the Eligibility code and they're both in the same DLL as the code that parses the iMAX file. As a developer, when I'm in the middle of making changes to only the EOI code, I don't want to have to comment out my changes so that I can check in changes to Elig code since EOI processing has nothing to do with Elig processing.

 

Success criteria: There should be no changes to the EOI or the Elig files after processing an iMAX file.
